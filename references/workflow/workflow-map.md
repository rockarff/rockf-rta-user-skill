# Workflow Map

The RTA system should be read as one pipeline:

1. Discovery
2. User
3. Persona
4. Content

Meaning:

- Discovery decides what kind of evidence can be collected
- User decides who the client should serve
- Persona decides how the client should speak
- Content decides what to publish next

More specifically:

- Discovery must first decide whether enough evidence can be collected for downstream-customer portraits
- User must preserve closed / near-close / unfit portrait directions
- Persona should speak to the user types confirmed upstream
- Content should build broad-traffic and semi-vertical topics on those portrait directions, not on the client's business description alone
- Workflow should finally normalize all upstream outputs into one report-input package
