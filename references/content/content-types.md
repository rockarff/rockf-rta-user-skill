# 内容类型规则

RTA-CONTENT 默认将公域内容分成四类：

1. 广流量内容
2. 半垂直流量内容
3. 信任内容
4. 转化内容

`私域内容` 不再属于默认公域配比，只在用户明确要求时单独生成。

## 1. 广流量内容

目标：

- 让人停下来
- 让当前要打的那类具体用户画像本人先代入
- 进入这个具体画像本人本来就会停下来的关注轴
- 先讲这类人作为“人”本来就在意的身份、处境、压力、关系、位置感

前提判断：

- 不要把广流量理解成固定模板
- 先判断这个项目的广流量，应该从哪一种来源长出来
- 再决定题库方向，而不是先写题再硬归类

通常用于：

- 当前要打的那个具体用户画像本人会停留、共鸣、转发的话题
- 这个画像本人的身份、处境、焦虑、关系、压力、欲望、错位
- 例如：35 岁深圳男性律师，广流量可以从生意焦虑、专业与影响力错位、城市竞争压力、获客不稳定、线上表达起不来切入

对需要“广流量先进入、再往专业上收”的内容结构，尤其要注意：

- 广流量不是先讲服务
- 不是先讲方法
- 不是先讲“你该怎么做内容”
- 不是先讲“你的业务问题怎么解决”

而是先讲：

- 这类人最近为什么累
- 这类人最怕什么位置变化
- 这类人最容易在哪种关系里吃亏
- 这类人最容易在哪种压力下误判
- 这类人为什么会停下来

常见来源至少有 4 类：

1. 目标用户本人的现实处境
2. 目标用户所处的商业变化
3. 目标用户终端用户关心的话题
4. 目标用户的身份表达冲突

判断时要先回答：

- 这个项目当前更适合从哪一种来源起广流量
- 为什么是这一种，不是另外一种
- 这一种来源，和后面的半垂直、信任、转化怎么接得上

不要要求它直接承担专业证明和深转化。

也不要把某种方案预设成唯一答案。

不要一上来就写成：

- 为什么你一定要做内容
- 为什么你现在必须做个人 IP
- 为什么线上获客才是出路

更合理的顺序是：

1. 先讲真实困境
2. 先讲旧路径为什么失灵
3. 先讲判断错位在哪里
4. 最后再让方案作为可能解法之一出现

一句话：

`广流量先打“这个人为什么会停下来”，不是先打“这个问题怎么解决”。`

## 2. 半垂直流量内容

目标：

- 从泛兴趣往业务相关兴趣收口
- 让人意识到“这个人懂我，也懂我所在的行业或处境”
- 建立从广流量到专业内容的过渡带

通常用于：

- 把这个具体画像的身份和处境，往表达、信任、获客、个人经营收
- 用户专长对该画像的洞察
- 用户专长对该画像所在行业或工作处境的洞察
- 行业现象、身份场景、问题前兆、常见误判

半垂直不是“稍微垂直一点”。

它的真正任务是：

- 把前面的现实问题翻译到业务相关问题
- 把大问题往表达、信任、内容、获客、成交前认知上收
- 为后面的信任和转化铺路

如果说广流量讲的是：

- 这个人最近怎么了
- 这个人为什么会焦虑
- 这个人为什么会卡住

那半垂直讲的是：

- 这些焦虑为什么会落到表达、信任、获客上
- 这些卡点为什么不是情绪问题，而是经营问题
- 为什么很多人不是没能力，而是没把能力翻译出去

一句话：

`广流量先讲人，半垂直再讲这个人的问题为什么会落到业务与表达上。`

## 3. 信任内容

目标：

- 让用户确认你有真专长
- 让用户确认你不是只会讲外层现象
- 让用户愿意继续听你讲更深的判断

通常用于：

- 用户的专长内容
- 方法、判断、经验、处理逻辑
- 更纯的专业能力证明

## 4. 转化内容

目标：

- 让用户愿意咨询
- 让用户愿意继续沟通
- 让用户知道为什么现在要找你

通常用于：

- 核心用户
- 已经进入强需求区的用户

## 可选：私域内容

目标：

- 长期关系
- 关系维护
- 合作信任
- 转介绍

通常只在明确要求时用于：

- 高价值用户
- 线下关系
- 私域场景
