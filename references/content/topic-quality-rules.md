# 选题精准度规则

这份规则只解决一个问题：

为什么有些案例出的题更落地、更精准、更像案主本人，
而有些案例虽然结构没错，但就是“不像他”。

## 判断顺序

每次你感觉题不准，按这个顺序排查：

1. 用户画像准不准
2. 准备成交画像有没有
3. 案主原生母题有没有
4. 案主表达语料够不够
5. 母题是不是先于结构
6. 单题是否可拍

## 1. 用户画像不准，广流量一定会飘

如果画像只有：

- 老板
- 宝妈
- 律师
- 创业者

这种大词，不够。

必须尽量补到：

- 年龄段
- 城市层级
- 当前处境
- 当前压力
- 当前最在意的收益和损失
- 现在会停下来看什么

## 2. 没有准备成交画像，半垂直很难准

半垂直不是泛专业。

半垂直是：

- 这个准备成交的人
- 正在什么具体处境里
- 会被什么专业判断吸住

所以没有准备成交画像时，半垂直只能做成“方向对”，很难做到“击中”。

## 3. 没有案主原生母题，题库就会像代理写的

案主原生母题的标准是：

- 他会反复讲
- 他讲起来不费劲
- 他一开口就会往下展开
- 他对这个题有判断，不只是有知识

如果没有这层，题库容易变成：

- 业务上成立
- 表达上不属于他

## 4. 没有表达语料，问题就容易太整齐

题库太整齐，通常是因为只有结构，没有语料。

要尽量拿：

- 访谈逐字稿
- 客户常说的话
- 客户的判断句
- 客户的口头习惯

## 5. 先母题，后结构

结构永远不能替代母题。

正确顺序：

1. 内容层级
2. 母题
3. 五种结构
4. 具体问法

## 6. 单题必须通过“可拍性”检查

每条题至少过这 5 条：

1. 案主看到就知道怎么讲
2. 题里带真实场景
3. 有明确追问空间
4. 能逼出判断标准
5. 不需要再翻译成“人话”
