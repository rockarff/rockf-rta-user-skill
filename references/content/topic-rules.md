# 选题规则

## 选题必须绑定 5 个字段

每个选题都必须标注：

1. 用户类型
2. 生命周期阶段
3. 内容类型
4. 使用场景
5. 目标动作

## 选题不是孤立标题

不要只输出标题。

这里的“选题”默认应同时包含两层：

1. `短视频选题`
2. `访谈提问`

也就是说，不再只输出其中一类。

- 短视频选题：用于内容生产、口播标题、封面文案和选题筛选
- 访谈提问：用于案主展开表达、访谈拍摄、辅助脚本拆解

访谈提问优先输出：

- 为什么……
- 什么情况下……
- 根据你的经验……
- 你怎么看……
- 哪些征兆说明……

例如：

- 短视频选题：`很多老板把“再谈谈”当办法，其实是在给对方留时间`
- 访谈提问：`为什么你觉得老板心中的“再谈谈”，其实是在给被告留机会？`

- 短视频选题：`股东闹翻之前，通常都会先出现这几个小动作`
- 访谈提问：`根据你的经验，股东相互起诉之前有哪些小动作或者是征兆？`

至少补充：

- 选题方向
- 结尾钩子
- 评论区引导
- 目标动作

## 选题数量

默认输出范围：

- 最少 10 个
- 最多 50 个

优先根据内容地图分组输出。

如果用户没有特别限制，建议默认先输出：

- 10-20 个

只有在用户明确需要更大题库时，再扩到 30-50 个。

如果用户明确要求：

- “不少于 50 条”

则必须输出至少 50 条。

## 比例规则

如果用户明确给出内容比例，必须优先遵守。

例如：

- 广流量 50%
- 半垂直流量 20%
- 信任 10%
- 转化 10%

如果总数不能整除，优先做到最接近整数分配。

## 先判断，再出题

正式起题前，先做 4 步判断：

1. 先判断这个项目这一轮更需要什么内容结构和进入方式
2. 先判断它当前更缺停留，还是更缺靠近
3. 先判断广流量来源属于哪一类
4. 再判断半垂直要往哪里收

如果这 4 步没做，不要直接铺题库。

## 内容层级和出题结构是两套东西

先分清两件事：

1. `广流量 / 半垂直流量 / 信任 / 转化` 是内容层级
2. `判断题 / 场景题 / 误判题 / 代价题 / 立场题` 是出题结构

不要把它们混成一套。

正确顺序是：

1. 先定本轮内容层级
2. 再定每个层级下要打哪几个母题
3. 再把每个母题拆成不同结构

例如：

- `广流量` 不是一种题型
- `半垂直流量` 也不是一种题型
- 它们都可以继续拆成：
  - 判断题
  - 场景题
  - 误判题
  - 代价题
  - 立场题

如果没有这层拆分，很容易出现：

- 题目看起来很多
- 实际都在说同一件事
- 只是标题换词，不是信息变多

## 母题拆题规则

如果要批量生成题库，优先按：

`母题 × 多种结构`

去拆，而不是顺着一个业务点平铺很多近义标题。

默认至少拆这 5 种：

1. 判断题
2. 场景题
3. 误判题
4. 代价题
5. 立场题

举例：

母题如果是：

- `很多专业人士不是不专业，是线上表达没有被优先选择`

那么可以拆成：

- 判断题：`为什么很多专业人士明明专业不差，却一直没有被优先选择？`
- 场景题：`什么情况下，一个人专业很强，但客户还是不会先找他？`
- 误判题：`很多人把“没人来问”当成能力问题，真正的问题可能是什么？`
- 代价题：`如果一个专业人士一直没有被优先选择，后面最容易付出什么代价？`
- 立场题：`你同不同意，专业能力和被优先选择，本来就是两件事？`

默认要求：

- 同一个母题拆出来的 5 条题，信息重点必须不同
- 不是只换句式
- 不是只换几个词
- 必须换冲突、换视角、换信息增量

## 先选题，后出框架

默认流程：

1. 先给选题池
2. 用户确认选题
3. 再问是否需要辅助框架

辅助框架包括：

- 开头钩子
- 中段结构
- 结尾钩子
- 评论区引导
- 完整脚本

## 双列选题规则

每个选题默认要同时生成：

- 一条 `短视频选题`
- 一条 `开放提问`
- 一条 `追问`
- 一条 `判断提问`

其中：

- 短视频选题负责抓注意力和表达核心冲突
- 开放提问负责让案主先自己展开
- 追问负责把信息往具体场景、动作、征兆上带
- 判断提问负责逼出案主的判断标准和方法

开放提问优先满足这几个标准：

1. 题目本身是一个问题
2. 能自然引出案主经验、判断或案例
3. 题目里尽量保留真实业务场景
4. 不要预设太多前置结论
5. 要更像案主在访谈中会被问到的问题

追问优先问：

- 具体动作
- 具体场景
- 最早征兆
- 常见误判

判断提问优先问：

- 你怎么判断……
- 什么情况下你会警惕……
- 哪个节点你会建议……

推荐句式：

- 为什么你觉得……
- 根据你的经验……
- 什么情况下……
- 你一般怎么判断……
- 哪些动作/征兆意味着……
- 很多人以为……，你怎么看？

## 防止“炒冷饭”

判断“是不是重复”，不要只看标题字面。

真正要检查的是：

1. 这条题在打哪个母题
2. 它属于哪一种结构
3. 它带出的新信息是什么
4. 它和上一条相比，冲突点有没有变
5. 它和上一轮题库相比，用户停下来的理由有没有变

以下情况都算重复：

- 同一个意思换了近义词
- 同一个判断换了两个问法
- 同一个场景只改了主语
- 同一个结论，只是从“为什么”改成“是不是”

以下情况才算真正新增：

- 场景变了
- 判断标准变了
- 误判点变了
- 代价变了
- 立场冲突变了

## 精准度差异判断

如果你发现：

- 某个案例的题库更落地、更像案主本人
- 另一个案例虽然结构对，但“味道不对”

不要只改标题。

先判断差异出在哪里。

### 一、先看母题来源

题库不准，最常见的问题不是结构，而是母题不对。

要先分清：

1. 这是案主原生母题
2. 还是你根据资料替他归纳出来的业务母题

优先级永远是：

`案主原生母题 > 归纳业务母题`

如果没有原生母题，题库可以生成，但必须承认：

- 结构会对
- 方向可能也对
- 但不一定像他本人会讲的话

### 二、先看用户画像有没有具体到“准备成交画像”

如果用户画像只有：

- 行业
- 大概客群
- 模糊需求

那广流量和半垂直流量通常都会飘。

必须尽量拿到：

1. 已成交画像
2. 准备成交画像
3. 不成交画像

尤其是 `准备成交画像`。

因为它决定：

- 广流量到底打什么停留点
- 半垂直到底往哪一层专业收

### 三、先看案主有没有自己的表达语料

如果只有业务资料，没有案主说话语料，题库容易出现两个问题：

1. 结构正确但不像他本人
2. 问题成立但他讲不顺

所以起题前要判断：

- 这个母题是不是他会反复讲
- 他能不能自然展开 2-3 分钟
- 他有没有自己的判断和情绪

### 四、先看题是不是从“结论”倒推出来的

以下方式容易失真：

- 先有结论
- 再换 5 种问法

正确方式是：

1. 先定内容层级
2. 再定母题
3. 再拆结构
4. 再检查每条题是不是独立成立

### 五、先看题是不是“可拍”

真正好题库，不只是逻辑成立。

还要满足：

1. 案主一看就知道自己会怎么讲
2. 问题里有真实场景
3. 能自然追问
4. 能带出判断标准
5. 不需要再翻译一次才敢拍

### 六、碰到不准时的修正顺序

不要一上来改 50 条标题。

修正顺序固定是：

1. 先补用户画像，尤其准备成交画像
2. 再补案主原生母题
3. 再判断哪些母题适合公域主战场
4. 再重新拆五种结构
5. 最后才修单条问法

## 不要预设方案为唯一答案

选题不要提前把商业方案写死。

不要默认倒推这些结论：

- 你现在必须做内容
- 你一定要做个人 IP
- 线上获客一定比线下更重要
- 只有某一种服务路径才成立

更合理的出题顺序是：

1. 先讲真实困境
2. 先讲旧路径为什么失灵
3. 先讲误判和错位
4. 最后再让方案作为可能解法之一出现

如果一个选题读起来像销售结论，不像现实判断，就要重写。

## 访谈提问风格参考

访谈提问这一列，默认尽量贴近你的常用提问方式：

1. 可以用“好奇 / 发现”开头
   - 例如：`我有个好奇点……`
   - 例如：`我发现一个小洞察……`

2. 问题尽量短
   - 能短就短
   - 优先控制在一句话内
   - 如果能问到 10 个字左右，更好

3. 先接对方的话，再追问
   - 不是生硬换话题
   - 要有“你刚刚提到……那是不是可以理解为……”这种承接感

4. 多用数字、场景、动作替代抽象概念
   - 少问空泛概念
   - 多问：`哪一步？`
   - 多问：`什么时候？`
   - 多问：`出现了什么动作？`

5. 可以适当自曝短板，拉近距离
   - 例如：`这个我可能不太懂，你给我讲细一点`
   - 例如：`我之前了解过一点，但这里还是不太确定`

6. 小切口进入
   - 先从具体小事实问起
   - 再往深处追
   - 不要一上来就抽象总结

7. 少用封闭式“有没有 / 是不是”收口
   - 除非只是为了开场确认
   - 更好的方式是让对方自然展开

## 私域开关

如果当前客户明确不需要私域内容，本轮题库里不要生产私域选题。

## 公域默认结构说明

默认公域内容结构现在是：

1. 广流量内容
2. 半垂直流量内容
3. 信任内容
4. 转化内容

其中：

- 广流量：当前要打的那个具体用户画像本人，会停下来看的身份、处境、焦虑、关系、压力、欲望、错位
- 半垂直流量：把这些身份和处境，往表达、信任、获客、个人经营收
- 信任：用户的专长内容
- 转化：用户的转化内容

这里要特别注意：

- `广流量` 先打“这个人为什么会停下来”
- `半垂直流量` 再打“这个人的问题为什么会收回到业务、表达、判断、获客上”

不要一上来就把广流量写成：

- 服务介绍
- 业务解释
- 方法输出
- 解决方案推销

对需要“广流量先进入、再往专业上收”的内容结构来说：

- 广流量先讲人
- 半垂直再讲事

## 广流量来源判断

不要把广流量理解成一种固定模板。

至少先判断它更接近哪一种来源：

1. 目标用户本人的现实处境
2. 目标用户所处的商业变化
3. 目标用户终端用户关心的话题
4. 目标用户的身份表达冲突

举例：

- 顾问、教练、专业服务、高信任成交项目，常从“目标用户本人处境”起
- 创业者、服务商、B 端转型项目，常从“商业变化”起
- 宝妈、家庭、教育、情绪、消费型 IP，常从“终端用户话题”起
- 律师、医生、财税顾问、保险顾问等强专业身份项目，常从“身份表达冲突”起

题库开始前，必须先判断：

- 当前最适合的广流量来源是哪一类
- 这一类为什么和后续半垂直能接上

## 广流量起题顺序

后面做题库时，默认顺序应该是：

1. 先抽象成稳定画像层
2. 再定本轮重点具体画像
3. 拆这个画像当前最强的关注轴
4. 用这些轴长广流量
5. 再把这些轴往内容、表达、获客收成半垂直流量

所谓“先抽象成稳定画像层”，指的是不要一上来只抓一个过近的样本身份。

优先先抽这些更稳定的标签：

- 一二线城市 / 下沉城市
- 男性 / 女性
- 25-35 / 30-40 / 40+
- 专业人士 / 创业者 / 服务者 / 管理者
- 高客单 / 低客单
- 高信任成交 / 低信任成交
- 强熟人关系生意 / 强公开流量生意

然后再回到本轮要打的具体画像。

这样长出来的题更稳，也更适合一个账号持续打同一类人。

常用关注轴：

- 身份
- 压力
- 竞争
- 生意
- 信任
- 位置感
- 责任
- 体面
- 稳定感
- 被选择
- 专业与影响力错位

## 阶段使用规则

### 1-6 阶段

更适合做内容选题，尤其是短视频。

可以重点使用：

- 典型场景
- 用户想法
- 判断信号

### 7-10 阶段

更适合做：

- 咨询承接
- 私域内容
- 转介绍内容
- 合作维护内容

原则上不作为短视频主选题区。
