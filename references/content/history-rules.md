# 历史记录规则

RTA-CONTENT 生成选题时，不只要出结果，还要记录历史。

## 每次生成必须记录

至少记录：

1. 客户名称
2. 第几次生成
3. 生成日期
4. 本次选题总数
5. 本次内容比例
6. 本次完整题库文档路径

## 建议目录

```text
outputs/history/{client_name}/
├── index.json
├── generation-001-topic-library.md
├── generation-002-topic-library.md
└── ...
```

## 新旧题库差异规则

“不一样”指的是：

- 选题标题本身不一样

不算“不一样”的情况：

- 只是换了问法
- 只是改了几个字
- 只是把语序前后调换

## 默认动作

新一轮生成前：

1. 先读取该客户历史题库
2. 提取历史标题
3. 避免重复标题
4. 再生成新一轮题库
