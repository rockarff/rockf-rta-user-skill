# Quickstart Prompts

这是给客户安装完以后直接复制使用的极简话术。

## 1. 我不知道现在该从哪一步开始

```text
请使用 RTA skill。
先判断我现在处于哪一个阶段。
告诉我下一步该做什么，如果前置资料不够，请先补问最关键的问题。
```

## 2. 我是新客户，还没有完整资料

```text
请使用 RTA skill。
我现在是一个新客户，请先从 Discovery 开始。
先判断我属于零基础、已验证还是混合型，再给我访谈稿和资料清单。
```

## 3. 我已经有访谈逐字稿

```text
请使用 RTA skill。
我已经有访谈逐字稿，请判断我现在能不能直接进入 User 阶段。
如果信息不够，请先告诉我还缺什么。
```

## 4. 我已经有用户画像和用户生命旅程

```text
请使用 RTA skill。
我已经有用户画像和用户生命旅程，请判断能否进入 Persona 和 Content。
如果还不能，请明确告诉我缺的输入。
```

## 5. 我想直接做报告交付

```text
请使用 RTA skill。
我希望把当前已有的 RTA 结果整理成一个交付包。
请先判断是否够生成报告，如果不够，请明确指出缺失项。
```

## 为什么这样写

因为客户第一次使用时，最容易出的问题不是“不会提问”，而是：

- 不知道该从哪一步开始
- 资料不够却直接想做后面的事
- 跳过人工确认闸门

这几句 prompt 的目的就是先把流程站稳。
