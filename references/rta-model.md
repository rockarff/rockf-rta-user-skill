# RTA Model

RTA = Reach - Trust - Asset.

One-line definition:

`Personal IP is not a content project. It is a personal business model where the person is first seen, then trusted, and finally converted into an asset.`

## Reach

Reach solves:

`Why do you have ability, but the market has not seen you quickly enough?`

Key parts:

- positioning
- basic profile setup
- expression theme
- traffic content

## Trust

Trust solves:

`Why do people see you, but do not continue to trust you?`

Key parts:

- user profile
- user lifecycle
- trust content
- user operation

RTA-USER focuses mainly on this layer.

## Asset

Asset solves:

`Why do people trust you, but the trust has not become a long-term asset?`

Key parts:

- transaction design
- delivery
- proof
- repurchase and referral

## Larger Loop

Use all outputs for the larger loop:

`content -> user -> business`

User insights are not the final goal. They must become usable input for later content, product, sales, and delivery work.
