# Client Types

## 1. `zero_based`

Use when most of these are true:

- no stable paying clients
- no clear winning user segment
- no reliable sales case review
- no meaningful content feedback data
- business direction still exploratory

Core logic:

`Do not ask who already buys. Ask who they are most likely to help, persuade, and attract first.`

## 2. `validated`

Use when most of these are true:

- has paying clients
- has real sales cases
- can describe recent customers
- can describe why customers bought
- can describe good-fit and bad-fit clients

Core logic:

`Use real customer and transaction evidence first.`

## 3. `mixed`

Use when both are partly true:

- some real customers exist, but no stable pattern yet
- some content or consulting exists, but not enough to trust fully
- some direction is validated, but the next phase is still exploratory

Core logic:

`Use what is real, then extend with forward-looking judgment.`

## Minimum Classification Questions

1. Do you already have paying clients?
2. Can you describe 2-3 recent sales or service cases?
3. Do you already create content?
4. Do you know which kind of customer you want most?
5. Are you trying to refine an existing direction or start from scratch?
