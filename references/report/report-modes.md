# Report Modes

RTA-REPORT supports two modes.

## full_report

Use when the upstream chain is mostly complete:

- Discovery exists
- user profiles exist
- user lifecycle exists
- persona exists
- content judgment or content map exists

This mode should produce a client-facing report that can be sent directly after a final visual pass.

## stage_report

Use when the upstream chain is incomplete, but there is still enough material for a stage delivery.

Typical cases:

- only User is complete
- User and Persona are complete, but Content is not
- Persona is provisional

This mode must clearly state:

- what stage is complete
- what is not complete
- what the report can conclude now
- what the next step should be
