# Fixed Report Template

这个 skill 的默认模板已经固定为项目内验证通过的版本。

以后默认复用：

- 同一套结构
- 同一套信息密度
- 同一套视觉方向

除非用户明确要求改风格，否则不要重新发明版式。

## 模板目标

交付一份：

- 像顾问公司阶段报告
- 不是 AI 笔记排版
- 不是 PPT 文字堆砌
- 结构固定、表达克制、判断前置

## 固定主报告结构

### 1. Cover

固定元素：

- 标题
- 副标题
- 阶段标签
- 右侧摘要判断带

要求：

- 标题必须直接是报告名
- 副标题只能说明材料来源和用途
- 不能写空泛漂亮话

### 2. 用户画像总表

固定元素：

- 总表
- 四类用户卡片

要求：

- 先总览，后细分
- 细分卡片必须包含 identity / scenes / surface / deep / trust / signals / judgment

### 3. 用户生命旅程使用规则

固定元素：

- 生命周期使用规则 chips
- 10 阶段横轴

要求：

- 必须明确标出前 6 阶段是公域主选题区
- 7-10 阶段是咨询、服务、转介绍区

### 4. 关键生命周期拆解

固定元素：

- 按用户类型拆表

要求：

- 字段固定为 stage / scene / thought / pain / signal
- 只放对内容和转化有用的阶段

### 5. 表达与人设判断

固定元素：

- 核心人设句
- 信任角色
- 表达优势
- 不适合表达

要求：

- 一句话必须能直接定义此人的表达角色
- 不适合表达必须明确写出

### 6. 内容判断总表

固定元素：

- 公域重点
- 内容比例
- 是否做私域
- 内容矩阵表

要求：

- 比例必须明确
- 不做什么也必须明确

### 7. 代表选题样本

固定元素：

- 选题卡片，不是表格

要求：

- 主报告只放代表样本，不放全量题库
- 每张卡必须展示：
  - 选题标题
  - 主问题
  - 开放追问
  - 深入追问
  - 判断追问

默认数量：

- 15 条
- 广流量 8
- 半垂直流量 4
- 信任 2
- 转化 1

### 8. 下一步动作

固定元素：

- 30 天动作
- 90 天动作

要求：

- 必须可执行
- 不能写空泛总结

## 固定附录结构

### 全量选题附录

固定为表格：

- ID
- 内容类型
- 对应用户
- 生命周期阶段
- 短视频选题
- 访谈提问

## 固定视觉方向

- Swiss data-report
- 浅底
- 蓝色信息层级
- 柠绿高亮只用来标阶段标签和数量标签
- 强信息密度
- 弱装饰

## 禁止事项

- 不要临时改成另一种版式
- 不要加入 AI 感很强的口号页
- 不要把主报告做成大段文字说明
- 不要把代表选题渲染成宽表格
