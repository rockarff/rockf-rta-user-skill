# Visual Rules

The report should feel like a business research report, not a text memo.

The default validated direction is:

- Swiss data-report
- restrained business palette
- no decorative AI-style slogans
- dense information blocks with short labels

Use one primary visual per page:

- cover
- summary cards
- layered user cards
- timeline
- decision-stage ladder
- stage matrix
- persona card
- trust map
- content matrix
- do / do not list
- roadmap
- interview-card topic sample
- appendix topic table

Rules:

1. one page, one conclusion
2. visual first, supporting text second
3. avoid paragraphs longer than three lines
4. use color to distinguish stage or content role, not just decoration
5. every visual must support a business recommendation
6. representative topics in the main report must use interview-card layout
7. full topic libraries must live in a separate appendix, not in the main report body
