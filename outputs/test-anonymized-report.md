# RTA-USER 匿名示例报告

## 一、业务背景

- 客户类型：专业服务型个人 IP
- 当前业务：以咨询、顾问、陪跑为主
- 当前客户来源：熟人介绍、朋友圈、零散内容咨询
- 当前问题：内容有输出，但没有稳定承接到咨询与成交
- 本次目标：先做清楚用户是谁、用户怎么一步步走到成交

---

## 二、四类用户画像

### 1. 潜在用户

**用户名称**  
方向模糊的起步者

**一句话描述**  
已经意识到自己需要做内容，但还没想清楚该怎么讲、讲给谁听。

**用户身份**  
顾问、教练、律师、咨询型个体从业者

**当前处境**  
有一些专业能力，但表达没有体系，也没有稳定咨询来源。

**典型需求**

- 想知道自己该怎么表达
- 想找到更适合自己的内容切口
- 想知道内容和成交之间怎么连接

**表层问题**

- 不知道拍什么
- 发了内容没人问
- 内容方向来回变

**深层痛点**

- 怕自己走错方向
- 怕花了很多时间也没结果
- 怕自己不适合做个人 IP

**触发场景**

- 看到同行通过内容接咨询
- 线下来源开始减少
- 想转型但没有稳定路径

**决策顾虑**

- 怕方法不适合自己
- 怕买了之后还是不会做
- 怕只是泛泛而谈

**信任来源**

- 真实案例
- 对自己行业表达的理解
- 能把模糊问题讲清楚的人

**付费意愿**  
低到中

**适配判断**  
适合先做验证型合作，不适合一开始就上高强度长期陪跑。

---

### 2. 核心用户

**用户名称**  
已有业务基础的表达升级者

**一句话描述**  
已经有专业能力和部分成交，希望把表达系统化，提升稳定咨询质量。

**用户身份**  
有稳定业务的个人专业服务者

**当前处境**  
已经有客户，但内容与业务没有打通，表达偏零散。

**典型需求**

- 梳理用户画像
- 梳理用户生命旅程
- 建立后续内容框架

**表层问题**

- 内容和业务脱节
- 咨询质量不稳定
- 用户来了但承接不清楚

**深层痛点**

- 怕内容做成自嗨
- 怕自己讲不出差异
- 怕持续做也无法提升成交质量

**触发场景**

- 已经开始重视个人 IP
- 希望放大专业影响力
- 希望提升更优质客户比例

**决策顾虑**

- 怕交付不落地
- 怕分析不够懂业务
- 怕只停留在“内容建议”

**信任来源**

- 能准确复述自己的业务问题
- 有真实成交理解
- 能清晰拆出后续动作

**付费意愿**  
中到高

**适配判断**  
最适合进入完整前期交付。

---

### 3. 高价值用户

**用户名称**  
要搭建长期增长系统的决策者

**一句话描述**  
不只想做内容，而是想把内容、承接和后续运营都逐步系统化。

**用户身份**  
高客单专业服务负责人 / 个人品牌决策者

**当前处境**  
已有业务验证，但增长还依赖个人经验，不够可复制。

**典型需求**

- 明确高价值用户从哪里来
- 明确内容如何进入业务
- 明确后续团队或长期运营方向

**表层问题**

- 内容产能不稳定
- 高价值用户路径不清晰
- 团队协同弱

**深层痛点**

- 怕增长不可复制
- 怕团队接不住
- 怕自己离开系统就失效

**触发场景**

- 开始搭团队
- 想做长期 IP
- 想提升高客单客户占比

**决策顾虑**

- 怕项目太轻
- 怕没有长期价值
- 怕内容和业务仍然断开

**信任来源**

- 真实高价值案例
- 对用户和商业都理解
- 能承接长期合作路径

**付费意愿**  
高

**适配判断**  
适合衔接长期陪跑、系统搭建、私域和运营协同。

---

### 4. 不合适用户

**用户名称**  
只想马上拿流量的人

**一句话描述**  
只想快速出结果，不愿意做用户研究和表达打磨。

**用户身份**  
短期投机型客户

**当前处境**  
焦虑、急、希望照抄一个答案。

**典型需求**

- 立刻涨粉
- 立刻出单

**表层问题**

- 过度关注表面数据
- 急于求成

**深层痛点**

- 没有耐心搭系统
- 不接受验证周期
- 不愿意配合交付过程

**适配判断**  
不适合进入当前合作模型。

## 三、十阶段用户生命旅程

### 阶段 1：无意识

- 典型场景：业务还靠熟人介绍，内容只是偶尔发
- 用户想法：可能不是平台问题，是我没讲清楚？
- 核心痛点：
  - 不知道问题出在哪
  - 没有判断标准
  - 说不清内容和成交的关系
- 判断信号：
  - 开始承认自己表达有问题
  - 愿意停下来听完整个观点
  - 会说“好像在讲我”

### 阶段 2：有意识

- 典型场景：内容持续没反馈，开始怀疑自己方式不对
- 用户想法：我可能真的需要有人帮我梳理
- 核心痛点：
  - 不知道从哪改
  - 不懂什么叫有效用户画像
  - 害怕自己判断失真
- 判断信号：
  - 开始问具体怎么做
  - 会收藏类似内容
  - 开始比较不同方法

### 阶段 3：记住你

- 典型场景：反复刷到你的内容
- 用户想法：这个人讲得挺准
- 核心痛点：
  - 信息太多记不住谁
  - 很多表达看起来都差不多
  - 难区分谁是真的懂业务
- 判断信号：
  - 能复述你的观点
  - 记住你的标签
  - 对你的表达有熟悉感

### 阶段 4：关注你

- 典型场景：开始翻看你的历史内容
- 用户想法：我再看看他后面怎么讲
- 核心痛点：
  - 内容太碎
  - 看不出完整体系
  - 不知道哪条最关键
- 判断信号：
  - 关注账号
  - 连续看多条内容
  - 会翻历史内容

### 阶段 5：了解你

- 典型场景：准备咨询前认真做功课
- 用户想法：他到底交付什么，适不适合我
- 核心痛点：
  - 怕交付模糊
  - 怕自己阶段不对
  - 怕买完还是不会做
- 判断信号：
  - 开始问服务边界
  - 开始比较合作路径
  - 会问具体交付细节

### 阶段 6：接触你

- 典型场景：私信、加微信、预约访谈
- 用户想法：我先聊聊，看他懂不懂我
- 核心痛点：
  - 不知道怎么描述自己
  - 怕被标准化推销
  - 怕聊了也没结果
- 判断信号：
  - 愿意说具体问题
  - 愿意提供背景材料
  - 愿意进入访谈

### 阶段 7：好朋友

- 典型场景：深聊后感受到被理解
- 用户想法：这个人确实懂我
- 核心痛点：
  - 怕被误解
  - 怕问题被讲浅了
  - 怕自己的真实诉求没被接住

### 阶段 8：选择你

- 典型场景：对比方案、确认合作
- 用户想法：先从哪一步开始更合适
- 核心痛点：
  - 预算焦虑
  - 怕第一步选错
  - 怕投入后执行不了

### 阶段 9：有收获

- 典型场景：拿到交付并开始使用
- 用户想法：终于知道问题卡在哪
- 核心痛点：
  - 怕拿到后不会用
  - 怕只是一份文件
  - 怕没有后续动作

### 阶段 10：转介绍

- 典型场景：主动推荐给朋友或同行
- 用户想法：这个方法也适合别人
- 核心痛点：
  - 怕推荐出去别人体验不好
  - 怕说不清价值
  - 怕结果不稳定

## 四、这份交付能怎么接后续动作

这类报告后面一般接三件事：

1. 做内容选题前置判断  
   用阶段 1-6 的典型场景、用户想法做选题前提

2. 做短视频结尾钩子  
   用阶段 1-6 的判断信号做结尾触发

3. 做后续人设与表达  
   用用户画像里的深层痛点、信任来源、决策顾虑来反推 IP 表达方式
